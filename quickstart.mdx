SlideMenu gets you a fully gesture-driven side menu with a few lines of SwiftUI. This guide walks you from a bare installation to a complete, customized side menu — including a toolbar toggle button, direction configuration, and appearance modifiers.

## Minimal working example

The simplest `SlideMenu` takes two trailing closures: `main` for your primary content and `menu` for the side panel.

```swift
import SlideMenu
import SwiftUI

SlideMenu {
    Text("Main Content")
} menu: {
    VStack {
        Text("Home")
        Text("Settings")
    }
}
```

This gives you a left-edge swipe gesture that reveals the menu. Swipe from the left edge or tap the dimmed overlay to close it.

## Adding a toggle button

Place a `SlideMenuButton` anywhere inside the `main` content. It reads the toggle action from the environment automatically — no bindings or callbacks required.

```swift
SlideMenu {
    NavigationStack {
        Text("Hello")
            .toolbar {
                ToolbarItem(placement: .topBarLeading) {
                    SlideMenuButton(systemImage: "sidebar.leading")
                }
            }
    }
} menu: {
    Text("Menu")
}
```

<Note>
`SlideMenuButton` works because `SlideMenu` injects a `slideMenuToggle` closure into the SwiftUI environment of both content trees. Any descendant view can read `@Environment(\.slideMenuToggle)` directly.
</Note>

## Custom button label

You can use any view as the button label:

```swift
SlideMenuButton {
    Label("Menu", systemImage: "line.3.horizontal")
}
```

## Choosing the direction

Use the `direction` parameter to slide from the trailing edge instead:

```swift
SlideMenu(direction: .right) {
    Text("Main")
} menu: {
    Text("Right Side Menu")
}
```

## Configuring the edge zone

The `edgeZoneWidth` parameter controls how far from the screen edge a swipe must begin in order to open the menu. The default is `30` points.

```swift
SlideMenu(edgeZoneWidth: 50) {
    // ...
} menu: {
    // ...
}
```

Increase this value if users have trouble triggering the swipe. Decrease it to reduce conflicts with horizontal controls near the edge.

## Full example with customization

Here's a complete example combining all parameters and appearance modifiers:

```swift
SlideMenu(direction: .left, edgeZoneWidth: 40) {
    NavigationStack {
        ContentView()
            .toolbar {
                ToolbarItem(placement: .topBarLeading) {
                    SlideMenuButton(systemImage: "sidebar.leading")
                }
            }
    }
} menu: {
    MenuView()
}
.slideMenuWidth(300)
.slideMenuBackground(.ultraThinMaterial)
.slideMenuOverlayOpacity(0.5)
.slideMenuBorderWidth(0)
.slideMenuAnimation(.bouncy(duration: 0.5, extraBounce: 0.15))
```

<Tip>
All view modifiers (`slideMenuWidth`, `slideMenuBackground`, etc.) are passed through the SwiftUI environment, so you can apply them to any ancestor view — not just `SlideMenu` itself.
</Tip>
